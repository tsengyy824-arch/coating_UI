# Modbus TCP/IP 模拟器测试指南

## 快速开始

### 步骤 1: 启动 Modbus 服务器模拟器

打开**第一个**终端窗口，运行：

```powershell
(.venv) PS C:\Users\eason.tseng\Desktop\0226> python modbus_server_simulator.py
```

你应该看到：
```
==================================================
機械臂 Modbus TCP/IP 模擬伺服器啟動
==================================================
服務器地址: 127.0.0.1
服務器端口: 502
...
```

### 步骤 2: 启动机械臂控制应用程序

打开**第二个**终端窗口，运行：

```powershell
(.venv) PS C:\Users\eason.tseng\Desktop\0226> python main.py
```

应用程序 GUI 窗口应该会出现。

### 步骤 3: 测试连接

1. 在 GUI 中点击「連接」按钮
2. 连接标签应该变成「已連接」（绿色）
3. 其他按钮（啟動、停止、塗膠等）应该变为可用

## 模拟器功能说明

模拟器支持以下功能：

| 类型 | 地址范围 | 说明 | 初始值 |
|------|--------|------|--------|
| 線圈 (Coils) | 0-99 | 数字输出，可读写 | OFF (False) |
| 離散輸入 | 0-99 | 数字输入，只读 | OFF (False) |
| 保持暫存器 | 0-99 | 模拟输出，可读写 | 0 |
| 輸入暫存器 | 0-99 | 模拟输入，只读 | 0 |

## 配置文件调整

如果端口 502 无法使用（权限问题），可以修改：

### 选项 A: 使用其他端口

编辑 `config/config.ini`：
```ini
[MODBUS]
PORT=5020
```

然后编辑 `modbus_server_simulator.py`，找到这一行：
```python
address=("127.0.0.1", 502),
```

改为：
```python
address=("127.0.0.1", 5020),
```

### 选项 B: 改回生产环境 IP

编辑 `config/config.ini`：
```ini
[MODBUS]
HOST=192.168.1.100
PORT=502
```

然后连接到真实的 IPC 设备。

## 常见问题

### Q: 模拟器启动失败，显示"Permission denied"或"Access is denied"

**A:** 端口 502 需要管理员权限。

解决方案：
- 以管理员身份打开终端
- 或使用选项 A 改为其他端口 (如 5020)

### Q: 按下「連接」按钮后显示"未連接"

**A:** 检查以下几点：

1. 模拟服务器是否仍在运行？ (第一个终端是否还有日志输出)
2. HOST 和 PORT 是否正确？ (查看 `config/config.ini`)
3. 防火墙是否阻止了连接？

### Q: 如何追踪连接和数据问题？

**A:** 查看终端日志：

- 服务器日志：**第一个终端**（模拟器日志）
- 客户端日志：**第二个终端**（应用程序日志）

所有操作都会打印详细的日志信息，用于调试。

## 测试场景

### 场景 1: 测试基本连接

1. 启动模拟器
2. 启动应用程序
3. 点击「連接」
4. 验证状态变为「已連接」

### 场景 2: 测试机械臂控制

1. 连接成功后，点击「啟動」按钮
2. 查看日志和状态显示
3. 点击「停止」按钮
4. 点击「斷開」断开连接

### 场景 3: 测试膠閥控制

1. 连接成功后，点击「塗膠 (ON)」按钮
2. 查看膠閥状态变为「塗膠中」
3. 点击「停膠 (OFF)」按钮
4. 查看膠閥状态变为「停膠」

## 停止模拟器

在模拟器终端按 `Ctrl+C` 来停止服务器。

## 返回生产环境

当需要连接真实 IPC 设备时：

1. 编辑 `config/config.ini`，改 HOST 为实际 IPC 地址
2. 停止本地模拟器
3. 运行应用程序，连接到真实设备
