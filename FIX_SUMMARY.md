**版本**: 2.2

## 🔧 修復的問題

### 問題名稱
**塗膠路徑沒有得選，無法試用自動塗膠按鈕**

### 問題描述
- 選擇塗膠閥類型後，路徑下拉菜單為空
- 無法選擇任何塗膠路徑
- 自動運行按鈕無法啟用

### 根本原因
1. DRA 項目目錄（`C:\DRA\Projects`）在測試環境中不存在
2. 路徑下拉菜單初始化時被設置為無選擇狀態（`setCurrentIndex(-1)`）
3. 預設路徑備用方案未正確激活
4. 缺少路徑選擇事件處理器

---

## 💡 實施的解決方案

### 1. 增強 `load_glue_paths()` 方法
- 確保 DRA 目錄不存在時使用預設路徑
- 異常處理中也提供預設路徑備份
- 路徑下拉菜單初始化時預選第一個選項（`setCurrentIndex(0)`）

```python
# 優化前：setCurrentIndex(-1)  → 無選擇
# 優化後：setCurrentIndex(0)   → 自動選擇第一個
```

### 2. 增強 `on_valve_type_changed()` 方法
- 檢查路徑計數，如果為 0 重新加載
- 自動選擇第一個路徑
- 自動啟用自動運行按鈕

### 3. 新增 `on_path_selected()` 方法
- 監聽路徑選擇的改變
- 路徑選擇改變時檢查閥類型是否已選
- 同時選定時自動啟用運行按鈕

### 4. 連接路徑信號
```python
self.path_combo.currentTextChanged.connect(self.on_path_selected)
```

### 5. 增強 `change_mode()` 方法
- 切換到自動模式時重設路徑到第一個選項
- 確保用戶在切換模式後看到已選的路徑
